# Evals — fable-operator

Assert that the skill fires on **substantive reasoning** work (irreversible actions, diagnosis,
load-bearing quantitative claims, user pushback, high-stakes drafts, conclusion review) and does
**not** fire on casual chat or settled facts (rigor theater), nor duplicate the SDD phase skills
(`verify`/`debug`) or stack builds (route there instead). The capability scenario checks the
load-bearing moves: triage → name irreversibility → verify the key claim by re-derivation →
decompose into checkable claims → calibrate bins → adversarial pass → answer-first with risk.
