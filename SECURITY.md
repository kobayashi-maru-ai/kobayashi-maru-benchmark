# Security policy

## Scope

Security reports may cover the CLI, web application, model-provider adapters,
result ingestion, prompt-injection boundaries in the judge, and accidental
publication of credentials or private evaluation material.

## Reporting

Do not open a public issue for a vulnerability. Use the repository's
[private vulnerability reporting](https://github.com/kobayashi-maru-ai/kobayashi-maru-benchmark/security/advisories/new)
form and include reproduction steps, affected revision, and expected impact.

## Operational boundary

Kobayashi processes fictional text scenarios only. It must never be connected
to weapons, targeting systems, police operations, vehicles, or other physical
actuators. A benchmark result is not a deployment-safety certification.
